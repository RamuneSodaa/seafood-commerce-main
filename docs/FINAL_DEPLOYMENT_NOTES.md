# 正式商家后台部署说明

当前 Mac/Windows 双击脚本只用于本地演示和开发试用。

正式上线后，商家不应该运行本地脚本，也不应该接触数据库、Node.js、npm 或本地环境配置文件。

## 正式商家使用方式

正式商家后台应提供：

1. HTTPS 后台域名
2. 正式管理员账号
3. 生产数据库
4. 生产 API
5. 微信小程序正式配置
6. 完整的管理员登录和权限控制
7. 操作日志和安全审计

## 商家不需要做的事

正式上线后，商家不需要：

1. 安装 Node.js
2. 安装 npm
3. 启动本地数据库
4. 运行本地脚本
5. 处理 `.local/env` 文件
6. 执行 seed 或数据库同步

## 正式上线前必须确认

1. 后台域名已启用 HTTPS。
2. API 使用生产环境配置。
3. 数据库为生产数据库。
4. 管理员账号不是本地开发账号。
5. 支付配置使用正式商户信息。
6. 微信小程序 AppID、合法域名和后台接口匹配。
7. 库存、订单、商品、门店、优惠券等写操作都有权限保护。
8. 关键操作有操作日志。

## 本地账号说明

本地账号 `admin` 仅用于本地开发演示，不允许作为生产账号。
