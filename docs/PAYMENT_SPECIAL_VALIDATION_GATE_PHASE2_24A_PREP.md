# Phase 2.24A 微信支付专项 Gate 准备

阶段：Phase 2.23F 输出  
性质：支付专项前置 checklist，不修改支付代码

本文只为下一阶段微信支付准正式/真机专项验收做 gate 准备，不包含任何证书、私钥、API key 或真实支付密钥。

## 1. 支付专项开始前必须确认的环境

| 项目 | 检查要求 |
| --- | --- |
| 小程序 AppID | 确认与微信开发者工具、微信后台、后端配置一致 |
| 微信支付商户号 | 确认使用目标验收商户号 |
| 商户 API 证书/私钥 | 仅通过安全环境变量或密钥管理注入，不进入仓库 |
| API v3 key | 仅在安全环境配置，不写入日志或 review pack |
| 回调 HTTPS 域名 | 必须公网 HTTPS 可访问，且配置到微信支付后台 |
| 小程序合法域名 | API 域名必须加入微信小程序 request 合法域名 |
| API BASE URL | 小程序体验版/真机应指向准正式或正式 API |
| 测试环境性质 | 明确是正式商户、沙盒、准正式还是开发环境 |
| 小程序版本 | 明确是开发版、体验版还是正式版 |

## 2. 支付专项必须覆盖的路径

1. 创建订单。
2. 创建微信支付参数。
3. 前端调起支付。
4. 用户取消支付。
5. 用户支付成功。
6. 支付回调验签。
7. 重复回调幂等。
8. 支付成功订单状态流转。
9. 支付成功优惠券从 `LOCKED` 变 `USED`。
10. 支付失败/取消支付后订单仍为待支付。
11. 订单超时取消释放优惠券。
12. 金额以后端订单金额为准。
13. 前端金额不作为可信金额来源。
14. 支付回调金额必须与订单金额一致。

## 3. 支付专项禁止事项

```text
不改 migration。
不改 seed。
不改顾客端非支付 UI。
不重构支付。
不泄露证书、私钥、API key。
不把支付密钥写入 review pack。
不为了测试伪造支付成功。
不绕过回调验签。
```

## 4. 支付专项需要人工截图 / 记录

1. checkout 金额页。
2. 点击立即支付前的订单详情页。
3. 微信支付拉起页面。
4. 用户取消支付后的订单详情页。
5. 用户支付成功后的订单详情页。
6. 优惠券状态变化截图：
   - 支付前 `LOCKED`
   - 支付后 `USED`
7. 回调日志脱敏摘要：
   - 回调收到时间
   - 验签结果
   - 商户订单号
   - 金额是否一致
   - 幂等处理结果

## 5. 进入支付专项前 Gate

必须满足：

```text
Phase 2.23E 临时空库 migration + seed + smoke PASS
P0/P1/P2/P3 = 0/0/0/0
新人券 autoGrant PASS
下单锁券 / 取消释放券 PASS
Prisma validate PASS
API tsc PASS
小程序 build PASS
```

如果以上任一项失败，不建议进入支付专项。
