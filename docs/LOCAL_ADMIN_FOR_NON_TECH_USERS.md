# 给非技术同事的绿膳荟后台启动说明

这份说明给不熟悉终端命令的同事使用。

## 先分清两种版本

### 本地演示版

当前这些双击脚本属于本地演示版，需要这台电脑已经准备好：

1. Node.js / npm
2. 本地数据库
3. 本地环境配置文件
4. 项目源码

如果缺少其中任何一项，请联系开发同事，不要自己新建配置文件。

### 正式商家版

正式上线后，商家不需要运行脚本，也不需要安装 Node.js、npm 或数据库。

正式方式应该是：打开一个 HTTPS 后台网址，输入正式账号密码登录。

## Mac 用户怎么打开

在 Finder 里打开：

```text
项目文件夹/mac
```

推荐双击英文兜底入口：

```text
start-admin-mac.command
```

也可以双击中文入口：

```text
打开绿膳荟后台.command
```

## Windows 用户怎么打开

在文件资源管理器里打开：

```text
项目文件夹/windows
```

推荐双击英文兜底入口：

```text
start-admin-windows.bat
```

也可以双击中文入口：

```text
打开绿膳荟后台.bat
```

Windows 脚本已生成，但必须在 Windows 真机上完成最终双击验证后，才能作为 Windows 交付通过。

## 登录地址和账号

```text
地址：http://127.0.0.1:3001/login
账号：admin
密码：<legacy-local-password-removed>
```

这是本地开发账号，只能用于本机演示，不是线上正式账号。

## 怎么停止后台

Mac 双击：

```text
stop-admin-mac.command
```

或：

```text
停止绿膳荟后台.command
```

Windows 双击：

```text
stop-admin-windows.bat
```

或：

```text
停止绿膳荟后台.bat
```

## 页面打不开怎么办

1. 先双击停止后台。
2. 再双击打开后台。
3. 等窗口提示后台已启动后，再打开登录页。
4. 如果仍不行，请截图发给开发同事。

## 页面变成纯文字、没有样式怎么办

Mac 双击：

```text
repair-admin-mac.command
```

Windows 双击：

```text
repair-admin-windows.bat
```

然后重新双击打开后台。

## 端口被占用怎么办

1. 先双击停止后台。
2. 再双击打开后台。
3. 如果仍提示端口被占用，请截图发给开发同事。

不要随便关闭不认识的程序。

## 什么情况要联系开发同事

1. 提示缺少运行环境。
2. 提示缺少本地环境配置文件。
3. 多次启动仍打不开。
4. 端口占用一直无法解决。
5. 登录账号无法进入后台。
6. 页面一直没有样式。

## 不要做的事情

1. 不要删除数据库文件。
2. 不要删除 `.local/env` 文件。
3. 不要把本地环境配置文件发给别人。
4. 不要把本地开发账号当成线上正式账号。
