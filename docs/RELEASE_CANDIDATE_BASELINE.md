# 绿膳荟顾客端本地候选版基线

阶段：Phase 2.22B_release_candidate_baseline_and_customer_regression  
基线时间：2026-06-14 13:01:44 CST  
性质：baseline / regression-only

## 基线结论

当前版本建议冻结为“顾客端本地候选版”。它适合作为后续回归、migration 审查、支付真机验收和上线准备的可回退参考状态。

这不是正式上线版，不是正式支付验收版，也不是生产数据库迁移版。

## 当前顾客端功能清单

- 品牌首页和商品列表。
- 商品分类图标、分类筛选、商品搜索。
- 商品卡、商品详情、SKU 底部弹窗。
- 加入购物车、购物车数量加减、移除商品、合计金额。
- 立即购买和购物车结算进入 checkout。
- checkout 商品明细、门店选择、到店自提、邮寄发货入口。
- 会员价展示和金额联动。
- 新人自动发券、优惠券中心、可用/锁定/已用/过期状态。
- checkout 单张券、多张券、全用、不用券、兑换码输入。
- 满减券差额提示和凑单推荐。
- 提交订单、待支付订单、订单详情。
- 待支付订单取消，取消后释放锁定优惠券。
- 待支付订单 lazy expiration 说明。
- 再来一单，支持合并加入购物车和清空购物车后加入。
- 我的页、会员中心、邀请分享 path 带 inviteCode。

## 当前已知未上线阻塞项

1. 正式 Prisma migration 未生成、未审查、未预发布演练。
2. 正式支付链路未在准生产/真机环境完整验收。
3. 生产 API 域名、HTTPS、微信合法域名、支付回调域名未确认。
4. 生产环境变量、证书、私钥、支付密钥管理未完成。
5. 小程序体验版、隐私协议、服务类目、审核材料未整理。
6. 后台运营能力仍需完整验收，尤其商品上下架、库存调整、订单处理、优惠券配置。
7. 当前项目目录不是 git 仓库，正式上线前需要建立版本基线或完整备份。

## 当前不建议继续新增功能的原因

- 顾客端核心闭环已经达到本地候选状态，继续加功能会扩大回归面。
- 正式上线真正阻塞在 migration、支付、部署、小程序审核和后台运营验收。
- 优惠券、会员价、凑单、订单取消释放券已经涉及金额和状态流转，应优先稳定而不是扩展。

## 基线用途

- 作为 Phase 2.22C 修复 P0/P1 问题的对照。
- 作为 Phase 2.23A 正式 migration 审查前的源码参考。
- 作为 Phase 2.24A 支付专项验收前的顾客端候选状态。
- 作为后续变更 review pack 和人工截图回归的稳定锚点。
