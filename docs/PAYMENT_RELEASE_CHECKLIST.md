# 微信支付正式上线检查清单

阶段：Phase 2.22A  
性质：检查清单，不改支付代码。

## 配置检查

| 检查项 | 当前判断 | 上线要求 |
| --- | --- | --- |
| 小程序 AppID | 已在本地修正过 dist/source 配置 | 正式发布前再次核对微信后台 AppID |
| 微信支付商户号 | 代码支持读取环境配置 | 必须使用正式商户号 |
| API v3 key / 证书 / 私钥 | 不在仓库中 | 必须只通过安全环境变量/密钥管理注入 |
| 支付回调 HTTPS 域名 | 待正式部署 | 必须配置微信支付可访问 HTTPS 域名 |
| 回调原始 body | 代码中读取 `rawBody` | 部署框架必须保留 raw body |
| 回调验签 | 已有微信平台公钥验签 helper | 必须准生产真机验证 |

## 业务检查

| 场景 | 要求 | 状态 |
| --- | --- | --- |
| 创建支付参数 | 只能当前顾客自己的待支付订单创建 | 需真机复测 |
| 金额来源 | 以后端订单金额为准 | 已按后端定价设计，需复测 |
| 前端金额可信度 | 前端金额不可被信任 | 已在文档中明确 |
| 支付成功 | 订单状态流转为已支付/待备货或待发货 | 需真机复测 |
| 支付成功用券 | LOCKED -> USED | 本地接口层已验证，需真机回归 |
| 支付失败/取消支付 | 订单仍待支付 | 需真机复测 |
| 待支付超时 | 超时取消并释放券 | 当前为 lazy expiration，正式建议定时任务 |
| 重复回调 | 幂等，不重复扣库存/用券 | 必须压测/准生产验证 |
| 金额不一致 | 拒绝或人工处理 | 必须补压测用例 |

## 必须在正式或准正式环境验证

1. 微信开发者工具真机支付调起。
2. 支付成功回调验签。
3. 回调重复发送幂等。
4. 回调金额与订单金额一致校验。
5. 支付成功后订单详情刷新。
6. 支付成功后优惠券 USED。
7. 支付取消后订单仍待支付。
8. 30 分钟超时后订单关闭和优惠券释放。

## 当前不改动

本阶段不修改微信支付签名、商户号、证书、私钥、回调核心逻辑。
